# Gate 3 — Words first (copy)

> Copy **before** design (the Dweik Method differentiator). Every page is drafted here,
> approved, then handed to Gate 4 (design/build). One file per page.

## How we write each page
1. **Skeleton** — section blocks already laid out in each file (Hero → problem → who → services → proof → exit → CTA, per WEBSITE_BUILDER_GUIDE section logic).
2. **Draft** into each `[TODO]` straight from:
   - `../copy-research/Language of the Customer.docx` (keywords, pains, desires, USE/AVOID)
   - `../copy-research/market-research.md` (segments + verbatim quote bank)
3. **Voice rules (non-negotiable):**
   - Speak **with** the hustle / War-Room worldview — never anti-guru, never down to them.
   - Plain English, punchy, skimmable, second person. Fast deciders.
   - Counter AI: "Claude can write your cold emails; it shouldn't file your taxes."
   - Lead with **the leak + the exit**; "keep more of what you make."
   - **Real content, no lorem** (builder guide). No "coming soon."
4. **Review → approve per page → lock** (flip status ⬜ → ✅ at top of the file).
